# 需求分析与结构化提取

本文件为 **fe-codegen-workbench** 流程第 2 步的详细说明。

## 分析流程

```
用户输入
  ├─ 纯文本需求 → 文本解析
  ├─ 原型图/设计稿 → 图片分析 + 文本补充
  └─ 接口文件/类型定义 → 字段提取
      ↓
结构化需求清单
```

## 1. 文本需求解析

从用户文本中提取以下信息：

| 提取项 | 识别关键词 | 示例 |
|--------|-----------|------|
| 页面类型 | 列表、表单、详情、弹窗、抽屉 | "做一个列表页" → 列表页 |
| 功能点 | 搜索、新增、编辑、删除、导入、导出 | "支持搜索和批量删除" |
| 字段列表 | 用户提供的接口结构或字段描述 | {"name": "姓名"} |
| 业务规则 | 默认值、限制、校验、权限 | "文件不超过5MB" |
| 交互方式 | 弹窗/抽屉/跳转新页面 | "点击查看跳转详情页" |
| 页面数量 | 用户描述的页面个数 | "包含列表、详情、编辑三个页面" |

## 2. 原型图分析

当用户提供原型图/设计稿/截图时：

### 分析要点

1. **布局结构**：页面整体布局（上下/左右/卡片/Tab）
2. **组件识别**：表格、表单、按钮、搜索栏、标签页、弹窗
3. **字段提取**：表头名称、表单标签、输入框类型
4. **交互线索**：按钮文字、操作列内容、状态标签
5. **样式特征**：颜色、间距、字体大小（如需还原设计稿）

### 分析输出模板

```
从原型图分析得出：
- 页面类型: 标准列表页（搜索 + 表格 + 操作）
- 搜索区域: 3 个搜索条件（姓名-输入框, 状态-下拉, 日期-范围选择）
- 表格字段: 序号, 姓名, 工号, 部门, 状态, 创建时间, 操作
- 操作列: 查看, 编辑, 删除
- 工具栏: 新增按钮, 批量导出按钮
- 弹窗/跳转: 点击编辑打开弹窗表单
```

## 3. 字段智能分类

根据字段名称/类型自动判断组件和搜索方式：

| 字段特征 | 判定类型 | 表格展示 | 搜索组件 | 表单组件 |
|----------|----------|----------|----------|----------|
| id/序号 | ID | 序号列 | - | - |
| name/名称/标题 | 文本 | 直接展示 | Input | Input |
| code/编号 | 文本 | 直接展示 | Input | Input |
| status/type/状态/类型 | 枚举 | Tag 标签 | Select | Select/Radio |
| amount/price/数量 | 数字 | 数字展示 | InputNumber 范围 | InputNumber |
| createTime/date/日期 | 日期 | 格式化展示 | DateRangePicker | DatePicker |
| isValid/enabled | 布尔 | Switch/Tag | Select | Switch |
| description/备注 | 长文本 | Tooltip 展示 | - | TextArea |
| departmentId/userId | 关联 | 关联名称 | 远程搜索 | Select 远程 |
| file/attachment/附件 | 文件 | 链接/预览 | - | Upload |

## 4. 页面类型判断

### 模板匹配规则

| 需求特征 | 匹配模板 | 关键标识 |
|----------|----------|----------|
| 搜索+表格+CRUD | 标准列表页 | "列表"/"表格"/"增删改查" |
| 新增/编辑弹窗（<10字段） | 弹窗表单 | "弹窗"/"对话框" |
| 新增/编辑抽屉（10-20字段） | 抽屉表单 | "抽屉"/"侧边" |
| 独立新增/编辑页面（>20字段） | 独立表单页 | "跳转"/"新页面"/"独立" |
| 查看详情-快速预览 | 抽屉详情 | "快速查看"/"侧边详情" |
| 查看详情-完整信息 | 独立详情页 | "详情页"/"审批流程" |
| Excel 数据导入 | 导入弹窗 | "导入"/"excel"/"xlsx" |
| 文件/图片上传 | 上传组件 | "上传"/"附件"/"图片" |
| 虚拟列表/大数据 | 大数据渲染 | "虚拟列表"/"8000条" |

### 多页面拆分

当需求涉及多个页面时，拆分为独立的需求单元。

## 5. 输出格式

分析完成后输出结构化需求清单（含模板 ID，与 [component-registry.json](./component-registry.json) 中的注册表及 MCP `match_template` 对齐）。

## 需要用户确认的决策点

1. 页面类型不确定（弹窗 vs 新页面）
2. 字段不完整
3. 业务规则模糊
4. 设计稿缺失细节
