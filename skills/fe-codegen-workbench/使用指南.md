# AI 驱动的前端页面开发指南（Skills 版）

> 适用于 PC 端 + H5 移动端页面开发，通过 Skills + 模板提示词（需求 + 原型描述 + 原型图）快速生成企业级页面

---

## 概述

基于 **fe-codegen-workbench** Skill 快速生成企业级前端页面的完整流程指南。通过 Skills 将代码生成规范、组件模板、技术栈知识库内置到 AI 编码工具中，无需额外配置即可生成高质量的可运行代码。

### Skills 版核心优势

| 特性 | 纯 AI 对话 | Skills 版 |
|------|-----------|----------|
| 编码规范 | AI 自由发挥 | 内置 code-standards，强制执行 |
| 模板匹配 | 无 | 16+ 种模板自动匹配 |
| 技术栈知识库 | 无 | React/AntdPro/Vue 最佳实践 |
| 代码审查 | 无 | code-review-expert 深度审查 |
| 跨工具兼容 | 绑定特定工具 | Claude Code / Codex / Cursor 通用 |
| 代码可用度 | 低-中 | **高** |

---

## 完整开发流程

```
用户输入（需求 + 原型描述 + 原型图）
    ↓
[1] 环境检测 → 有项目：检测技术栈 / 无项目：默认创建 React
    ↓
[2] 需求分析 → 结构化需求清单（页面类型、字段、业务规则）
    ↓
[3] 模板匹配 → 从 component-registry.json 匹配模板
    ↓
[4] 页面生成 → 加载技术栈知识库 → hooks 优先 → 组件 → 样式
    ↓
[5] 代码审查 → code-review-expert 审查 → 自检报告
```

> 以上步骤由 AI Agent 自动执行，用户只需提供输入。

## 第三步：AI 自动执行

提交提示词后，AI Agent 自动按以下流程执行：

### 1. 环境检测

#### 已有项目

```
检测结果：
- 项目路径: D:/project/xxx
- 技术栈: React + TypeScript
- UI 库: Ant Design Pro
- 全局类型: UserInfo, OrderItem（不可 import）
- 知识库已加载: react-antdpro-knowledge.md
```

#### 空目录（新建项目）

```
检测结果：
- 项目路径: D:/project/new-app
- 状态: 空目录，无 package.json
- 操作: 询问用户确认后创建项目
  → 默认: React + TypeScript + Ant Design Pro
  → 可选: Vue 3 + TypeScript + Element Plus
```

AI 会依次执行：
1. 确认技术栈选择
2. 创建项目骨架（Vite + 依赖安装）
3. 生成标准目录结构（pages/components/hooks/utils/types）
4. 继续后续的页面生成流程

### 2. 需求分析

AI 解析提示词/原型图，输出结构化需求清单。

### 3. 模板匹配

根据需求关键词 + 技术栈，从注册表匹配模板：

| 需求特征 | 匹配模板 |
|----------|---------|
| 搜索 + 表格 + CRUD | `react-standard-list-crud` |
| 弹窗新增/编辑 | `react-standard-modal-form` |
| 抽屉编辑 | `react-drawer-form` |
| 独立详情页 | `react-nonstandard-detail` |
| Excel 导入 | `react-import-list-modal` |

### 4. 页面生成

按强制顺序生成文件：

```
1. types.ts          ← 类型定义
2. hooks/*.ts        ← 业务逻辑（优先）
3. components/*.tsx  ← UI 组件
4. index.tsx         ← 页面入口
5. index.less        ← 样式
```

输出自检报告：

## 字段类型自动识别

AI 根据字段名称自动判断组件类型：

| 字段特征 | 表格展示 | 搜索组件 | 表单组件 |
|----------|---------|---------|---------|
| name / 名称 / 标题 | 直接展示 | Input 模糊搜索 | Input |
| status / type / 状态 | Tag 标签 | Select 下拉 | Select / Radio |
| createTime / date | 格式化日期 | DateRangePicker | DatePicker |
| amount / price | 数字格式化 | InputNumber 范围 | InputNumber |
| description / 备注 | Tooltip | — | TextArea |
| file / 附件 | 链接/预览 | — | Upload |

## 核心规则

1. **hooks / composables 优先**：先生成数据处理层，再生成 UI 层
2. **全局类型禁止 import**：`global.d.ts` 中的类型直接使用
3. **禁止 mock 数据**：不生成任何假数据
4. **接口数据以文件为主**：不推测字段
5. **技术栈知识库驱动**：根据检测到的技术栈自动加载对应最佳实践

## 使用建议

### 提升生成质量的技巧

1. **提供原型图**：AI 可以从截图中提取布局、字段、交互等信息
2. **引用已有接口**：比 JSON 数据结构更精确
3. **提供参考页面**：在提示词中加 `参考 xxx 页面的代码风格`
4. **明确业务规则**：校验规则、默认值、权限控制等

### 推荐使用模式

| 模式 | 代码可用度 | 适用场景 |
|------|-----------|---------|
| Skills + 需求描述 | 中 | 快速原型 |
| Skills + 需求 + 原型图 | 中高 | 标准页面 |
| Skills + 需求 + 原型图 + 接口文件 | 高 | 正式开发 |
| Skills + 需求 + 原型图 + 接口文件 + 参考页面 | **最高** | 精确还原 |
