# AI 驱动的前端页面开发指南（Skills 版）

> 适用于 PC 端 + H5 移动端页面开发，通过 Skills + 模板提示词（需求 + 原型描述 + 原型图）快速生成企业级页面

---

## 概述

基于 **fe-codegen-workbench** Skill 快速生成企业级前端页面的完整流程指南。通过 Skills 将代码生成规范、组件模板、技术栈知识库内置到 AI 编码工具中，无需额外配置即可生成高质量的可运行代码。

### Skills 版核心优势

| 特性 | 纯 AI 对话 | Skills 版 |
|------|-----------|----------|
| 编码规范 | AI 自由发挥 | 内置 code-standards，强制执行 |
| 模板匹配 | 无 | 16+ 种模板自动匹配 |
| 技术栈知识库 | 无 | React/AntdPro/Vue 最佳实践 |
| 代码审查 | 无 | code-review-expert 深度审查 |
| 跨工具兼容 | 绑定特定工具 | Claude Code / Codex / Cursor 通用 |
| 代码可用度 | 低-中 | **高** |

---

## 完整开发流程

```
用户输入（需求 + 原型描述 + 原型图）
    ↓
[1] 环境检测 → 有项目：检测技术栈 / 无项目：默认创建 React
    ↓
[2] 需求分析 → 结构化需求清单（页面类型、字段、业务规则 + designSignal）
    ↓
[2.5] 设计推荐 → 新建任务默认进入：主动推荐 3 个品牌 + "不使用品牌" 选项
    ↓  显式品牌/路径 → 跳过对话直接用
    ↓  显式"按默认" / 增量修改任务 → 跳过 2.5
    ↓  用户选"不使用品牌" → selectedBrandId=null，走默认样式
    ↓  用户选品牌 → 记录 selectedBrandId，进入 3.5
[3] 模板匹配 → 从 component-registry.json 匹配模板
    ↓
[3.5] DESIGN.md 加载（仅当 selectedBrandId 非空）→ 远程优先抓 detailUrl
    ↓
[4] 页面生成 → 加载技术栈知识库 → hooks 优先 → 组件 → 样式
    ↓  若选了品牌：追加生成主题层 src/theme/* + 项目根 DESIGN.md
[5] 代码审查 → code-review-expert 审查 → 自检报告（选了品牌时追加主题合规审查）
```

> 以上步骤由 AI Agent 自动执行，用户只需提供输入。步骤 2.5 会以 `ask_question` 形式弹出选择，用户一秒可选"不使用品牌"退出。

## 第一步：准备提示词

### 提示词四要素

模板提示词由以下四部分组成：

| 要素 | 必选 | 说明 |
|------|------|------|
| **@fe-codegen-workbench + 一句话约束** | 是 | 引用本 Skill，并用一句话强调按 skill 约束执行 |
| **文件夹名称** | 可选 | 生成的页面目录名（kebab-case） |
| **接口数据结构** | 是 | AI 自动判断：已有接口文件 / JSON 数据结构 / 混合模式 |
| **参考页面** | 可选 | 项目中已有页面路径，用于对齐代码风格与组织方式 |
| **页面需求** | 是 | 功能描述 + 页面组成 + 字段 + 业务规则 + 原型图（可选） |

### 提示词要素说明

#### @fe-codegen-workbench + 一句话约束

在提示词开头引用本 Skill。AI 会自动加载：
- 编码规范（hooks 优先、全局类型规则、文件生成顺序）
- 组件注册表（16+ 种模板自动匹配）
- 技术栈知识库（React/AntdPro 或 Vue/ElementPlus）
- 代码审查（code-review-expert 集成）

```
@fe-codegen-workbench 严格按照 skill 约束生成前端页面
```

推荐在 `@` 后面直接补一句简短约束，既保留简洁，也能提醒 AI 优先按 skill 调度执行，例如：
- `@fe-codegen-workbench 严格按照 skill 约束生成前端页面`
- `@fe-codegen-workbench 严格按照 skill 约束做一个列表页`

## 第三步：AI 自动执行

提交提示词后，AI Agent 自动按以下流程执行：

### 1. 环境检测

#### 已有项目

```
检测结果：
- 项目路径: D:/project/xxx
- 技术栈: React + TypeScript
- UI 库: Ant Design Pro
- 全局类型: UserInfo, OrderItem（不可 import）
- 知识库已加载: react-antdpro-knowledge.md
```

#### 空目录（新建项目）

```
检测结果：
- 项目路径: D:/project/new-app
- 状态: 空目录，无 package.json
- 操作: 询问用户确认后创建项目
  → 默认: React + TypeScript + Ant Design Pro
  → 可选: Vue 3 + TypeScript + Element Plus
```

AI 会依次执行：
1. 确认技术栈选择
2. 创建项目骨架（Vite + 依赖安装）
3. 生成标准目录结构（pages/components/hooks/utils/types）
4. 继续后续的页面生成流程

### 2. 需求分析

AI 解析提示词/原型图，输出结构化需求清单（含设计风格信号 `designSignal`）。

### 2.5 设计推荐（新建任务默认启用）

根据 `designSignal` 和任务类型决定是否弹出设计选择：

| 场景 | 行为 |
|------|------|
| 新建页面 / 新建项目 + 无显式品牌 + 未显式跳过 | ⭐ **默认弹 ask_question**：3 个推荐品牌 + "不使用品牌"兜底 |
| 用户显式说 _"用 Linear 风格做 XX"_ | 跳过对话，直接用该品牌 |
| 用户提供本地 DESIGN.md 路径 | 跳过对话，直接读本地文件 |
| 用户显式说 _"按默认 / B 端标准 / 不用设计系统"_ | 跳过 2.5，走 AntD 默认 |
| 增量修改任务（改已有页面） | 跳过 2.5，沿用现有主题 |

**推荐加载：远程优先 + 本地降级**

```
🌐 先抓 https://getdesign.md/ 最新品牌列表 → 覆盖本地 index.json
   ↓ 失败
💾 用本地缓存（加 warning "远程不可达，使用本地缓存"）
   ↓ 失败
❌ 静默跳过设计链路，走 AntD 默认
```

**用户可用指令（随时输入）**：
- `刷新设计库` / `更新设计库` - 主动抓远程覆盖本地
- `列出设计库` - 展示所有品牌 + 预览色块
- `离线模式` / `强制使用缓存` - 跳过远程只用本地
- `回到默认` - 删除已生成的主题层文件

### 3. 模板匹配

根据需求关键词 + 技术栈，从注册表匹配模板：

| 需求特征 | 匹配模板 |
|----------|---------|
| 搜索 + 表格 + CRUD | `react-standard-list-crud` |
| 弹窗新增/编辑 | `react-standard-modal-form` |
| 抽屉编辑 | `react-drawer-form` |
| 独立详情页 | `react-nonstandard-detail` |
| Excel 导入 | `react-import-list-modal` |

### 3.5 DESIGN.md 加载（仅当用户选了品牌）

远程优先抓取对应品牌的 DESIGN.md（覆盖本地 `references/design-systems/<brand-id>/DESIGN.md`），解析出最小必需字段：`color.primary / color.background / font.sans / radius.md / spacing.md` 等。解析失败时逐字段降级默认值。

### 4. 页面生成

按强制顺序生成文件：

```
1. types.ts                 ← 类型定义
2. hooks/*.ts               ← 业务逻辑（优先）
3. components/*.tsx         ← UI 组件
4. index.tsx                ← 页面入口
5. index.less               ← 样式
───────────────────────────  以下仅当选了品牌时生成（仅一次）
6. src/theme/token.ts       ← 主题层（React）
```

## 核心规则

1. **hooks / composables 优先**：先生成数据处理层，再生成 UI 层
2. **全局类型禁止 import**：`global.d.ts` 中的类型直接使用
3. **禁止 mock 数据**：不生成任何假数据
4. **接口数据以文件为主**：不推测字段
5. **技术栈知识库驱动**：根据检测到的技术栈自动加载对应最佳实践
6. **业务代码零硬编码视觉**：色值/字体/spacing 统一走主题层 token（即使选了品牌，业务代码仍不感知具体品牌）
7. **设计推荐远程优先**：2.5 / 3.5 一旦启用，索引和 DESIGN.md 都优先从 getdesign.md 实时抓取，断网才退本地缓存

## 使用建议

### 提升生成质量的技巧

1. **提供原型图**：AI 可以从截图中提取布局、字段、交互等信息
2. **引用已有接口**：比 JSON 数据结构更精确
3. **提供参考页面**：优先使用结构化字段 `参考页面：src/pages/xxx/`，比散写一句“参考 xxx 页面的代码风格”更稳定
4. **明确业务规则**：校验规则、默认值、权限控制等

### 推荐使用模式

| 模式 | 代码可用度 | 适用场景 |
|------|-----------|---------|
| Skills + 需求描述 | 中 | 快速原型 |
| Skills + 需求 + 原型图 | 中高 | 标准页面 |
| Skills + 需求 + 原型图 + 接口文件 | 高 | 正式开发 |
| Skills + 需求 + 原型图 + 接口文件 + 参考页面 | **最高** | 精确还原 |
| Skills + 需求 + 指定品牌风格（`用 linear 风格`） | **最高（含视觉）** | 需要品牌化/高保真视觉时 |

### 何时需要选品牌风格

| 场景 | 建议 |
|------|------|
| 企业内部 B 端系统（后台管理、CRM、工单等） | **选"不使用品牌"**，走 AntD 默认最稳定 |
| 对外产品官网 / 登录页 / 营销页 | 可以选一个品牌风格，增强观感 |
| AI / 开发者工具 / SaaS 首页 | 建议选品牌，Linear / Vercel / Anthropic 都是好选择 |
| 数据大屏 / Dashboard | 可以选品牌（如 Stripe / Linear），数据层次感更好 |
| 已有成熟设计系统的公司 | 用本地 DESIGN.md 路径（`设计参考 ./docs/brand/DESIGN.md`） |
